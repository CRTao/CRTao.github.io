電腦安全課程的作業，解述CTF和基礎題型的入門。

感覺還蠻好玩的，這邊紀錄一下過程。

作業總共有六題，前三題教導CTF比賽的過程，後三題學習Buffer Overflow的觀念。

A. fildes

Source Code

很簡單。了解題目的目標是呼叫出 cat flag。

FLAG{INCredible_Bu7_It5_jus7_4_13egin}


B. you_should_read_manual

Source Code

也是很簡單。了解基本的數學運算式與random函數的基礎。

FLAG{1_C4NN07_L1V3_WO_M4Nual11111111111}


C. nasty_rules

Source Code

XOR Encryption中會用到的 XOR Key。

觀察 兩個password間的關係就行了。

FLAG{PU7_p4ren7he5es_N3x7_T1m3}


D. time_will_stop

這題沒有C Code。題目要求你解析binary file。

$ xxd ./time_will_stop

FLAG{Bin4RyANaly515_S0_EZZZZZZzzzzz}


E. agent_hacker

這題開始要用到Buffer Overflow的技巧。

需要讓其中的參數Overflow來重新引導return address。

Source Code

觀察我們要的目標，是要呼叫函式 print_flag()。

開始來找函式的位置。

$ objdump -M intel -d agent_hacker

得知函式位置在 0x0000099b 。

就可以利用python的pwn模組來寫overflow 程式。

FLAG{Y0U_sucH_Cr4Zy_Agen7_hack3R}


F. ret_shellcode

Source Code

這題沒有函數有system command了。所以我們要建立shellcode。

還好題目有提供name[0x100]給我們存需要的東西。

預想將shellcode存入至Global variable中，

然後藉由overflow “buff[0x10]”將位置指向 “name[0x100]”來呼叫shellcode。

首先先找name的位置。

一種方法是利用 printf(“%p”,name);

另一中方法是用 readelf -S ret_shellcode 來找 .data的區塊，

再用 gdb 去找分配的位置。

得知name[0x100]位置為 0x601060。

相差值就是buff的資料上限，hex(78-60)=dec(24)。
可以做pwn解題工具了。

FLAG{Th3_Migh7Y_5h311?_BaK4NA}


完成作業題型後，自己跑去找很多過去的例題，不管是校內的或是國際的。

感覺都很有趣，就是在爬程式碼找解答，很好玩。

只是很多都超級複雜，有的buffer不夠要重新導向，有的是很特別的呼叫方式等等。

平常都不太會用到 gdb，玩CTF幾乎都要看呢。

還有Leet駭客語真的酷，但能用的機會很少。
